## 报告安全漏洞

如果您发现了安全漏洞，**请不要通过 GitHub Issues 公开报告**。

### 报告方式

请通过以下方式私下联系我们：

- **邮箱**: security@example.com（请替换为实际邮箱）
- **GitHub Security Advisories**: [提交安全建议](https://github.com/jonykchen/tickpulse/security/advisories/new)

### 报告内容

请在报告中包含以下信息：

1. **漏洞描述**: 清晰描述发现的漏洞
2. **影响范围**: 说明漏洞可能造成的影响
3. **复现步骤**: 详细说明如何复现漏洞
4. **修复建议**: 如果有想法，可以提供修复思路
5. **发现者信息**: 您的姓名/GitHub 用户名（可选，用于致谢）

### 响应时间

- **确认收到**: 48 小时内
- **初步评估**: 7 天内
- **修复发布**: 视漏洞严重程度而定

## 安全最佳实践

### API 密钥保护

本应用支持配置 LLM API 密钥，请注意：

- 🔴 **不要** 在代码中硬编码 API 密钥
- 🔴 **不要** 将包含密钥的配置文件提交到版本控制
- ✅ 使用应用内置的设置页面配置密钥
- ✅ 密钥存储在本地 SQLite 数据库中，不会上传到云端

### 数据安全

- 所有数据存储在本地 SQLite 数据库中
- 不会主动上传任何用户数据
- 行情数据通过 HTTPS 加密传输

### 网络安全

应用仅访问以下域名获取行情数据：

- `*.eastmoney.com` - 东方财富
- `*.10jqka.com.cn` - 同花顺
- `*.sina.com.cn` - 新浪财经
- `*.qq.com` - 腾讯财经
- `*.dzh.com.cn` - 大智慧

## 已知安全限制

1. **本地存储**: 配置和数据存储在本地，未加密。如果您的电脑被他人访问，数据可能泄露。
2. **网络传输**: 行情数据通过公网传输，虽然使用 HTTPS，但仍存在中间人攻击风险。
3. **API 密钥**: LLM API 密钥存储在本地数据库，建议定期更换。
