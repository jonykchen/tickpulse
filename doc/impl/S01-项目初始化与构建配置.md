## 4. 核心改进对比 (§1.3 完整表格)

| 维度 | Chrome 扩展（原版） | Tauri 桌面版 v2.0 | 改进原因 |
|------|---------------------|-------------------|----------|
| 后台进程 | Service Worker（会被休眠） | Rust 永久进程 | 彻底解决秒级刷新可靠性 |
| CORS | host_permissions 绕过 | Rust 直连，无 CORS | 安全权限收窄 |
| 浮点计算 | JS Number（有精度误差） | rust_decimal | 金融计算零误差 |
| 并发请求 | JS 单线程 fetch | reqwest 连接池 + tokio 多线程 | 200只行情拉取效率提升10x |
| 存储 | chrome.storage.local + JSON | SQLite + LRU 内存缓存 | 事务支持、查询能力、版本迁移 |
| 交易日判断 | 简单 weekday 判断 | 交易日历 API + SQLite 缓存 | 准确识别节假日/调休 |
| 调度策略 | 固定10秒 | 14阶段MarketPhase精细化调度 | 集合竞价/午休/收盘竞价/盘后交易各不同 |
| 涨跌停 | 无 | 6板块差异化引擎 + 向上/向下取整 + 历史切换 | 主板/创业板/科创板/北交所/ST/退市各不同 |
| 收盘竞价 | 无差异 | 沪市主板连续竞价 / 深市+科创板收盘竞价 | 交易所规则差异 |
| 新股临停 | 无 | 沪市±10%/±20%，科创/创业±30%/±60% | A股新股交易规则 |
| 异动检测 | 无 | 快速拉升/下跌/量比突增/封板炸板/临停 | A股短线核心需求 |
| 行情推送 | 全量推送 | 增量推送（仅变化数据） | 减少IPC开销，降低CPU占用 |
| 数据容灾 | 单 fallback | 多级断路器 + 限速器 + 数据自愈 | 自动熔断+恢复+重建连接池 |
| 包体积 | ~2MB（扩展） | ~5MB（安装包） | 可接受范围 |
| 内存占用 | ~80MB（Chrome 进程树） | ~40MB（Tauri） | 全天运行场景敏感 |
