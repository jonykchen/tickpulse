# S03 — 交易日历模块
> 阶段：Phase 1 | 前置依赖：S02 | 来源：v2.0 §3.3

## 1. 概述

A 股节假日/调休无法通过 weekday 简单判断。交易日历模块负责准确判断某日是否为交易日，采用多级缓存策略：内存 → SQLite → API → weekday 降级。

### 4.3 is_trading_day() 完整判断链

判断路径：内存缓存 → SQLite → API → 降级 weekday

```rust
    /// 判断今天是否为交易日
    /// 路径：内存缓存 → SQLite → API → 降级weekday
    pub async fn is_trading_day(&self) -> bool {
        let today = Local::now().date_naive();
        if matches!(today.weekday(), Weekday::Sat | Weekday::Sun) { return false; }
        if self.trade_days.contains(&today) { return true; }

        // 尝试 SQLite → API → 降级
        match self.load_or_fetch(today).await {
            Ok(days) => { self.trade_days = days; self.trade_days.contains(&today) }
            Err(_) => true, // 降级：工作日视为交易日
        }
    }
```

```rust
    pub async fn is_trading_day_by_quote(&self, source: &dyn MarketDataSource) -> bool;
}
```

## 5. 测试要点

1. **周末判断**：周六、周日直接返回 false
2. **节假日判断**：国庆、春节等长假期间返回 false
3. **调休判断**：周六/周日调休上班时返回 true
4. **缓存命中**：内存缓存命中时跳过 API 请求
5. **降级逻辑**：API 失败时工作日默认视为交易日
6. **备用方法**：通过上证指数行情日期验证交易日
