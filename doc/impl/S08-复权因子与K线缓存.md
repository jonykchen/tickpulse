# S08 — 复权因子与K线缓存
> 阶段：Phase 2 | 前置依赖：S03(行情调度器) | 来源：v2.0 §3.9 + §3.10

## 1. 概述

本模块负责本地复权因子计算和 K 线分层缓存策略。v2.0 新增：本地复权因子计算，用于 K 线增量合并时跨越除权日的对齐；K 线缓存按周期和交易时段设置差异化 TTL，支持增量更新与复权对齐。

核心职责：
- 除权除息信息结构化存储
- 复权因子计算（前复权/后复权）
- K 线缓存分层 TTL 策略（按周期 + 盘中/盘后）
- K 线增量合并（跨越除权日时复权对齐）

## 4. 核心逻辑

### 4.1 AdjustFactorCalculator — 复权因子计算器

```rust
/// 复权因子计算器
pub struct AdjustFactorCalculator;

impl AdjustFactorCalculator {
    /// 计算单次除权的复权因子
    /// 复权因子 = 除权后参考价 / 除权前收盘价
    pub fn calc_factor(exright: &ExRightInfo, prev_close: Decimal) -> Decimal {
        let denominator = Decimal::ONE + exright.bonus_share + exright.allot_share;
        if denominator.is_zero() { return Decimal::ONE; }
        let ex_price = (prev_close - exright.dividend + exright.allot_price * exright.allot_share)
            / denominator;
        if prev_close.is_zero() { return Decimal::ONE; }
        ex_price / prev_close
    }

    /// 前复权：从除权日向后累乘复权因子
    /// 前复权价 = 原始价 × ∏(后续每次除权的复权因子)
    pub fn forward_adjust(
        original_price: Decimal,
        factors_after: &[Decimal],  // 该日期之后所有除权因子（按时间正序）
    ) -> Decimal {
        let mut result = original_price;
        for &factor in factors_after {
            result = result * factor;
        }
        result
    }

    /// 后复权：从除权日向前累乘复权因子
    /// 后复权价 = 原始价 × ∏(之前每次除权因子倒数)
    pub fn backward_adjust(
        original_price: Decimal,
        factors_before: &[Decimal],  // 该日期之前所有除权因子
    ) -> Decimal {
        let mut result = original_price;
        for &factor in factors_before {
            if !factor.is_zero() { result = result / factor; }
        }
        result
    }
}
```

### 4.2 复权公式速查

```
除权参考价 = (股权登记日收盘价 - 每股派息 + 配股价×配股比例) / (1 + 送转股比例 + 配股比例)

前复权价 = 原始价 × 复权因子（从该日向后累乘）
后复权价 = 原始价 / 复权因子（从该日向前累乘）

东方财富API fqt参数：0=不复权，1=前复权，2=后复权
本地缓存增量合并时，若跨越除权日需用 AdjustFactorCalculator 重新对齐
```

### 4.3 KlineCachePolicy::ttl — 缓存TTL策略

```rust
impl KlineCachePolicy {
    pub fn ttl(period: &KlinePeriod, is_trading_hours: bool) -> Duration {
        match period {
            KlinePeriod::Min1 | KlinePeriod::Min5 | KlinePeriod::Min15 | KlinePeriod::Min30 =>
                if is_trading_hours { Duration::ZERO } else { Duration::from_secs(3600) },
            KlinePeriod::Min60 | KlinePeriod::Min120 =>
                if is_trading_hours { Duration::from_secs(300) } else { Duration::from_secs(3600) },
            KlinePeriod::Day =>
                if is_trading_hours { Duration::from_secs(30) } else { Duration::from_secs(86400) },
            _ =>
                if is_trading_hours { Duration::from_secs(300) } else { Duration::from_secs(604800) },
        }
    }
}
```

### 4.4 KlineUpdater::merge_incremental — K线增量合并

```rust
impl KlineUpdater {
    pub fn merge_incremental(
        cached: &mut Vec<KlineBar>,
        recent: Vec<KlineBar>,
        adjust_type: AdjustType,
        exrights: &[ExRightInfo],  // v2.0: 除权信息
    ) {
        for bar in recent {
            if let Some(existing) = cached.iter_mut().find(|b| b.time == bar.time) {
                *existing = bar;
            } else {
                // v2.0: 检查新增K线是否跨越除权日
                // 若跨越，需用 AdjustFactorCalculator 重新计算复权价
                cached.push(bar);
            }
        }
        cached.sort_by_key(|b| b.time);
    }
}
```

### 4.5 K线缓存TTL一览表

| 周期 | 盘中TTL | 盘后TTL | 更新方式 |
|------|---------|---------|----------|
| 1/5/15/30分钟 | 0 | 1小时 | 全量 |
| 60/120分钟 | 5分钟 | 1小时 | 增量 |
| 日K | 30秒 | 1天 | 增量（复权对齐） |
| 周/月/季/年K | 5分钟 | 7天 | 增量（复权对齐） |

## 5. 测试要点

| 测试场景 | 预期结果 |
|----------|----------|
| 10送10，前收盘 20.00 | 复权因子 = (20-0+0)/(1+1+0) / 20 = 10/20 = 0.5 |
| 10派5，前收盘 10.00 | 复权因子 = (10-0.5)/(1+0) / 10 = 9.5/10 = 0.95 |
| 10配2配股价8元，前收盘 10.00 | 复权因子 = (10-0+8×0.2)/(1+0+0.2) / 10 = 11.6/1.2/10 ≈ 0.9667 |
| 前复权：原始价 10.00，因子 [0.95, 0.9] | 结果 = 10 × 0.95 × 0.9 = 8.55 |
| 后复权：原始价 10.00，因子 [0.95, 0.9] | 结果 = 10 / 0.95 / 0.9 ≈ 11.6959 |
| 分母为零（bonus_share=allot_share=0） | calc_factor 返回 Decimal::ONE |
| prev_close 为零 | calc_factor 返回 Decimal::ONE |
| 盘中1分钟K线 TTL | Duration::ZERO（实时拉取） |
| 盘后1分钟K线 TTL | 3600 秒 |
| 盘中日K TTL | 30 秒 |
| 盘后日K TTL | 86400 秒 |
| 盘后周K TTL | 604800 秒 |
| 增量合并：已有K线更新 | existing 被替换 |
| 增量合并：新增K线 | push 并按 time 排序 |
| 增量合并：跨越除权日 | 需 AdjustFactorCalculator 重新对齐 |
