# S10 — 多数据源对接与容灾

> 阶段：Phase 3 | 前置依赖：S03 | 来源：v2.0 §3.8 + §6.2

## 1. 概述

本模块负责多数据源的统一管理与容灾机制。核心能力包括：

- **断路器（CircuitBreaker）**：基于失败计数的熔断保护，支持 Closed/Open/HalfOpen 三态切换
- **数据源管理器（DataSourceManager）**：按优先级尝试多个数据源，断路器保护，限速器控制请求频率
- **并行分批请求**：200 只股票拆分为 4 批 x 50 并行拉取
- **限速器（RateLimiter）**：防止请求过载

数据源方面，除主数据源东方财富外，还包括新浪财经（K线 fallback）、同花顺（热门股/涨跌分布）、天天基金（基金实时估值）、大智慧（涨停天梯）。

## 3. 数据结构

### 3.1 CircuitState — 断路器状态

```rust
#[derive(Debug, Clone, Copy, PartialEq)]
pub enum CircuitState { Closed, Open, HalfOpen }
```

### 3.2 CircuitBreaker — 断路器

```rust
pub struct CircuitBreaker {
    failure_count: AtomicU32,
    failure_threshold: u32,        // 默认3次
    open_duration: Duration,       // 默认30秒
    state: RwLock<CircuitState>,
    last_state_change: RwLock<Instant>,
}
```

**字段说明**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `failure_count` | `AtomicU32` | 当前连续失败次数 |
| `failure_threshold` | `u32` | 熔断阈值，默认 3 次 |
| `open_duration` | `Duration` | 熔断持续时间，默认 30 秒 |
| `state` | `RwLock<CircuitState>` | 当前断路器状态 |
| `last_state_change` | `RwLock<Instant>` | 最近一次状态变更时间 |

**状态机**：

- **Closed** → 正常放行请求；失败次数达 `failure_threshold` 时切换到 Open
- **Open** → 拒绝所有请求；经过 `open_duration` 后切换到 HalfOpen
- **HalfOpen** → 放行一次试探请求；成功则回 Closed，失败则回 Open

## 4. 核心逻辑

### 4.1 fetch_quotes — 按优先级尝试 + 断路器保护

```rust
impl DataSourceManager {
    /// 按优先级尝试，断路器保护
    pub async fn fetch_quotes(&self, secids: &[String]) -> Result<Vec<StockQuote>> {
        self.rate_limiter.acquire().await;
        for source in &self.sources {
            let breaker = self.get_breaker(source.name());
            if !breaker.allow_request().await { continue; }
            match source.fetch_quotes(secids).await {
                Ok(quotes) => { breaker.record_success().await; return Ok(quotes); }
                Err(e) => { breaker.record_failure(source.name()).await; }
            }
        }
        Err(anyhow!("所有数据源均不可用"))
    }
}
```

**执行流程**：

1. 通过 `rate_limiter.acquire()` 获取请求令牌
2. 遍历数据源（按优先级排序）
3. 检查对应断路器是否放行（`allow_request()`）
4. 断路器拒绝则跳过该数据源，尝试下一个
5. 请求成功：记录成功并返回结果
6. 请求失败：记录失败（可能触发熔断），继续尝试下一个数据源
7. 全部失败：返回 "所有数据源均不可用" 错误

## 5. 测试要点

1. **断路器状态机**：验证 Closed → Open（连续 3 次失败）、Open → HalfOpen（30 秒后）、HalfOpen → Closed（试探成功）、HalfOpen → Open（试探失败）
2. **数据源优先级**：模拟主数据源失败，验证自动切换到备用数据源
3. **全源不可用**：所有数据源断路器打开时，返回正确错误信息
4. **并行分批**：验证 200 只股票拆分为 4 批并行，结果正确合并
5. **限速器**：验证请求频率不超过设定阈值
6. **东方财富字段映射**：验证 f2/f3/f4/f8/f9/f10/f23 等字段正确映射到 StockQuote
7. **JSONP 解析**：天天基金返回 JSONP 格式，验证正确提取 JSON 数据
