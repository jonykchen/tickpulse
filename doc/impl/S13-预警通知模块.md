# S13 — 预警通知模块

> 阶段：Phase 5 | 前置依赖：S03, S07 | 来源：v2.0 §3.12

## 1. 概述

本模块负责预警通知功能，核心能力包括：

- **AlertType 12 种变体**：涵盖价格、涨跌幅、量比、换手率、涨跌停、异动、临停等预警类型
- **AlertManager**：预警规则管理器，负责检查行情数据、触发预警、发送通知
- **check_and_notify()**：遍历行情数据，匹配预警规则并触发通知
- **send_notification()**：通过系统 Toast 发送预警通知
- **reset_daily()**：每日开盘重置触发状态，避免重复通知

## 3. 数据结构

### 3.1 AlertType — 预警类型

```rust
#[derive(Debug, Clone, Serialize, Deserialize)]
pub enum AlertType {
    PriceAbove(Decimal), PriceBelow(Decimal),
    ChangeAbove(Decimal), ChangeBelow(Decimal),
    VolumeRatioAbove(f64), TurnoverRateAbove(f64),
    LimitUp, LimitDown,
    AnomalyRise, AnomalyFall,  // 异动预警
    TempSuspend,               // v2.0: 新股临停预警
}
```

**12 种变体详细说明**：

| 变体 | 参数类型 | 触发条件 | 说明 |
|------|----------|----------|------|
| `PriceAbove(Decimal)` | 价格阈值 | 最新价 > 阈值 | 价格上穿预警 |
| `PriceBelow(Decimal)` | 价格阈值 | 最新价 < 阈值 | 价格下穿预警 |
| `ChangeAbove(Decimal)` | 涨跌幅阈值 | 涨跌幅 > 阈值 | 涨幅超限预警 |
| `ChangeBelow(Decimal)` | 涨跌幅阈值 | 涨跌幅 < 阈值 | 跌幅超限预警 |
| `VolumeRatioAbove(f64)` | 量比阈值 | 量比 > 阈值 | 量比突增预警 |
| `TurnoverRateAbove(f64)` | 换手率阈值 | 换手率 > 阈值 | 换手率异常预警 |
| `LimitUp` | 无 | 涨停 | 涨停预警 |
| `LimitDown` | 无 | 跌停 | 跌停预警 |
| `AnomalyRise` | 无 | 快速拉升异动 | 异动上涨预警 |
| `AnomalyFall` | 无 | 快速下跌异动 | 异动下跌预警 |
| `TempSuspend` | 无 | 新股临时停牌 | 新股临停预警（v2.0） |

## 4. 核心逻辑

### 4.1 check_and_notify — 检查并通知

```rust
impl AlertManager {
    pub fn check_and_notify(&mut self, quotes: &[StockQuote]);
}
```

**执行流程**：

1. 遍历所有已启用的预警规则（`enabled = true`）
2. 在行情数据中查找规则对应的股票（`secid` 匹配）
3. 根据 `AlertType` 变体判断是否触发：
   - `PriceAbove(threshold)`：`quote.price > threshold`
   - `PriceBelow(threshold)`：`quote.price < threshold`
   - `ChangeAbove(threshold)`：`quote.change_percent > threshold`
   - `ChangeBelow(threshold)`：`quote.change_percent < threshold`
   - `VolumeRatioAbove(threshold)`：`quote.volume_ratio > threshold`
   - `TurnoverRateAbove(threshold)`：`quote.turnover_rate > threshold`
   - `LimitUp`：`quote.is_limit_up == true`
   - `LimitDown`：`quote.is_limit_down == true`
   - `AnomalyRise`：依赖异动检测结果
   - `AnomalyFall`：依赖异动检测结果
   - `TempSuspend`：`quote.is_temp_suspended == true`
4. 触发时调用 `send_notification()`，标记 `triggered = 1`
5. 已触发的规则当日不再重复触发

### 4.2 send_notification — 发送通知

```rust
impl AlertManager {
    fn send_notification(&self, rule: &AlertRule, quote: &StockQuote);  // 系统Toast
}
```

通过 `tauri-plugin-notification` 发送系统原生 Toast 通知。

### 4.3 reset_daily — 每日重置

```rust
impl AlertManager {
    pub fn reset_daily(&mut self);  // 每日开盘重置触发状态
}
```

在每个交易日开盘时调用，将所有规则的 `triggered` 状态重置为 0，确保当日可重新触发预警。

## 5. 测试要点

1. **每种 AlertType 变体触发**：逐一验证 12 种预警类型的触发条件
2. **PriceAbove/PriceBelow**：价格恰好等于阈值时不触发，大于/小于时触发
3. **ChangeAbove/ChangeBelow**：涨跌幅精确匹配 Decimal 比较
4. **VolumeRatioAbove/TurnoverRateAbove**：f64 类型阈值比较
5. **LimitUp/LimitDown**：验证 `is_limit_up`/`is_limit_down` 标志触发
6. **AnomalyRise/AnomalyFall**：依赖异动检测引擎结果
7. **TempSuspend**：验证新股临停触发预警（v2.0）
8. **防重复触发**：已触发规则当日不再触发
9. **reset_daily**：重置后规则可重新触发
10. **系统 Toast**：验证 `tauri-plugin-notification` 正确发送通知
11. **SQLite 持久化**：规则 CRUD 正确，enabled/triggered 状态正确更新
