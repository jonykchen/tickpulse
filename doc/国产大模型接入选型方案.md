## 执行摘要

**核心结论**：推荐 **DeepSeek v4 + Qwen3.7** 作为第一梯队接入组合，GLM-5 为第二梯队。

| 梯队 | 模型 | 接入成本 | 单次分析成本 | 核心优势 |
|------|------|---------|------------|---------|
| 🥇 第一梯队 | **DeepSeek v4** | 极低（已适配） | ¥0.05-0.10 | 思维链可解释、已验证适配、极致性价比 |
| 🥇 第一梯队 | **Qwen3.7** | 低（OpenAI兼容） | ¥0.03-0.08 | 中文金融最强、结构化输出最稳定 |
| 🥈 第二梯队 | **GLM-5** | 低（OpenAI兼容） | ¥0.08-0.18 | 推理能力顶级、免费Flash兜底 |
| 🥉 第三梯队 | **MiniMax M3** | 低（需验证） | ¥0.01-0.03 | 月费包制极致低价 |
| ❌ 暂不推荐 | 百度文心/讯飞星火/百川 | 高（非标准API） | — | API不兼容、金融场景无优势 |

**推荐默认组合**：Quick-Think 用 `deepseek-v4-flash`，Deep-Think 用 `deepseek-v4-pro`，备选 Qwen3.7 系列。

## 1. 选型背景与目标

### 1.1 项目现状

TickPulse 是一款 A 股智能监控与分析桌面应用（Tauri + Rust + Vue），核心 AI 分析引擎已实现：

- **7 维度分析**：行业趋势 / 竞争格局 / 财务健康 / 管理层质量 / 成长性 / 估值 / 技术面
- **多空辩论**：Bull/Bear/ResearchManager 三方辩论 → 5 级评级输出
- **质量门控**：5 层硬检查 + LLM 复审（A-F 评级）
- **自动触发**：盘后 15:30 自动分析 + 异动触发（封板/炸板/量比突增）
- **决策记忆闭环**：T+1 反思 + 历史上下文注入

当前已支持的 LLM 供应商：

| 供应商 | 客户端 | 状态 |
|--------|--------|------|
| Anthropic (Claude) | `anthropic.rs` | ✅ 已实现 |
| OpenAI (GPT) | `openai_compat.rs` | ✅ 已实现 |
| DeepSeek | `openai_compat.rs` (兼容) | ✅ 已实现 |
| Ollama (本地) | `ollama.rs` | ✅ 已实现 |

### 1.2 为什么需要接入国产模型

1. **网络可达性**：Anthropic/OpenAI 在国内需代理，增加延迟和故障点
2. **合规与数据安全**：证券分析数据经境外传输存在合规风险
3. **成本优势**：国产模型价格普遍为海外模型的 1/10 ~ 1/50
4. **中文理解**：A 股特化 Prompt（T+1、涨跌停、ST、北向资金等）需强中文语义理解
5. **OpenAI 兼容**：项目已有 `openai_compat.rs`，国产模型大多兼容 OpenAI API 格式，接入成本极低

### 1.3 选型核心评估维度

| 维度 | 权重 | 说明 |
|------|------|------|
| **A 股场景适配** | 25% | 中文金融理解、A股规则认知、结构化分析能力 |
| **结构化输出** | 20% | JSON Schema / response_format 支持、评级解析可靠性 |
| **推理与思维链** | 15% | Reasoning/CoT 能力（多空辩论/质量门控复审依赖） |
| **API 兼容性** | 15% | OpenAI 兼容程度、reasoning_content 处理、tool_calls 支持 |
| **性价比** | 15% | 单次分析成本（7 维度 × 每维度 ~3k tokens 输出 + 辩论/门控） |
| **稳定性与生态** | 10% | SLA、并发限制、文档质量、社区活跃度 |

## 2. 候选模型详细评估

### 2.1 DeepSeek（深度求索）

#### 模型矩阵（2026 年最新）

| 模型 | 上下文 | 最大输出 | Thinking 模式 | 定价 (每百万 tokens) |
|------|--------|---------|--------------|---------------------|
| **deepseek-v4-flash** | 1M | 384K | ✅ 默认开启 | 输入 $0.14 (命中 $0.0028) / 输出 $0.28 |
| **deepseek-v4-pro** | 1M | 384K | ✅ 默认开启 | 输入 $0.435 (命中 $0.0036) / 输出 $0.87 |

> 注：`deepseek-chat` / `deepseek-reasoner` 将于 2026/07/24 下线，应直接使用 v4 系列。

#### 项目适配分析

| 评估项 | 评分 | 分析 |
|--------|------|------|
| A 股场景适配 | ⭐⭐⭐⭐⭐ | **已验证**：项目已实现 DeepSeek 适配（`reasoning_content` 回传、结构化输出降级），代码成熟度最高 |
| 结构化输出 | ⭐⭐⭐⭐ | v4-flash/pro 支持 JSON Output + Tool Calls；**注意**：Thinking 模式下不支持 `response_format`，需降级处理 |
| 推理与思维链 | ⭐⭐⭐⭐⭐ | Thinking 模式是核心卖点，reasoning_content 可用于质量门控复审的可解释性 |
| API 兼容性 | ⭐⭐⭐⭐⭐ | OpenAI 格式 + Anthropic 格式双兼容；项目已有 `messages_to_deepseek_format()` 适配 |
| 性价比 | ⭐⭐⭐⭐⭐ | v4-flash 极致性价比，单次分析约 ¥0.02-0.05；v4-pro 仍远低于 Claude/GPT |
| 稳定性与生态 | ⭐⭐⭐⭐ | 并发限制 v4-flash 2500 / v4-pro 500，足够个人/小团队使用；开源生态活跃 |

#### 接入成本

**极低** — 项目已完整实现 DeepSeek 适配，仅需将模型名从 `deepseek-chat` 迁移至 `deepseek-v4-flash` / `deepseek-v4-pro`。

#### 关键坑点（已在项目中处理）

1. `reasoning_content` 必须回传，否则 HTTP 400 → ✅ 已有 `adapt_request_for_deepseek()`
2. Thinking 模式下不支持 `tool_choice` / `response_format` → ✅ 已有 `supports_structured_output()` 降级
3. `deepseek-reasoner` 将下线 → 需迁移至 `deepseek-v4-pro`

#### 推荐用途

- **主选 Quick-Think 模型**：`deepseek-v4-flash`（7 维度分析 + 数据采集）
- **可选 Deep-Think 模型**：`deepseek-v4-pro`（综合决策 + 质量门控复审）

**低** — 仅需在 `CloudProvider` 枚举中新增 `Qwen` 变体，配置默认 base_url，复用 `openai_compat.rs` 即可。无需特殊格式适配。

```rust
// engine.rs 需新增
CloudProvider::Qwen => Box::new(openai_compat::OpenAICompatClient::new()),

// 默认 base_url
"https://dashscope.aliyuncs.com/compatible-mode/v1"
```

#### 推荐用途

- **首选 Quick-Think 模型**：`qwen3.6-flash`（7 维度分析，速度极快）
- **首选 Deep-Think 模型**：`qwen3.7-plus`（综合决策 + 辩论，性价比最优）
- **极致质量场景**：`qwen3.7-max`（当分析质量优先于成本时）

## 4. 推荐接入方案

### 4.1 双 LLM 模型推荐组合

基于项目的双 LLM 架构（Quick-Think + Deep-Think），推荐以下组合：

#### 🥇 方案 A：DeepSeek 双速组合（推荐默认）

| 角色 | 模型 | 单次分析成本 | 适用场景 |
|------|------|------------|---------|
| Quick-Think | `deepseek-v4-flash` | ¥0.02-0.05 | 7 维度分析、数据采集、简单判断 |
| Deep-Think | `deepseek-v4-pro` | ¥0.05-0.10 | 综合决策、多空辩论、质量门控复审 |

**优势**：已验证适配、思维链可解释、极低成本
**劣势**：Thinking 模式下不支持 `response_format`，需降级处理

#### 🥈 方案 B：Qwen 双速组合（推荐备选）

| 角色 | 模型 | 单次分析成本 | 适用场景 |
|------|------|------------|---------|
| Quick-Think | `qwen3.6-flash` | ¥0.01-0.03 | 7 维度分析、数据采集 |
| Deep-Think | `qwen3.7-plus` | ¥0.03-0.08 | 综合决策、多空辩论 |

**优势**：结构化输出最稳定、中文金融理解最强、阿里云 SLA
**劣势**：reasoning 能力略逊 DeepSeek v4-pro

#### 🥉 方案 C：跨供应商组合（推荐高级用户）

| 角色 | 模型 | 单次分析成本 | 适用场景 |
|------|------|------------|---------|
| Quick-Think | `qwen3.6-flash` | ¥0.01-0.03 | 维度分析（结构化输出最稳定） |
| Deep-Think | `deepseek-v4-pro` | ¥0.05-0.10 | 综合决策（推理能力最强） |

**优势**：各取所长，Quick 端用 Qwen 保证结构化输出质量，Deep 端用 DeepSeek 保证推理深度
**劣势**：双供应商管理复杂度略高

#### 🏅 方案 D：GLM 双速组合（推荐追求极致推理质量）

| 角色 | 模型 | 单次分析成本 | 适用场景 |
|------|------|------------|---------|
| Quick-Think | `glm-4.7-flashx` | ¥0.01-0.02 | 维度分析 |
| Deep-Think | `glm-5` / `glm-5.1` | ¥0.10-0.20 | 综合决策（推理对标 Claude Opus） |

**优势**：推理能力顶级、免费 Flash 兜底、1M 上下文可选
**劣势**：旗舰版定价略高、平台成熟度不及阿里云

### 4.2 单次分析成本预估

以项目"标准分析"预设为例（7 维度 + 辩论 + 门控）：

| 步骤 | 输入 tokens | 输出 tokens | 说明 |
|------|-----------|-----------|------|
| 7 维度分析 | 7 × ~2,000 = 14,000 | 7 × ~800 = 5,600 | 含 System Prompt + 数据 |
| 多空辩论 | ~4,000 | ~2,000 | Bull + Bear 各一轮 |
| 质量门控复审 | ~2,000 | ~500 | 仅失败维度触发 |
| 综合决策 | ~6,000 | ~1,500 | 含所有维度汇总 |
| **合计** | **~26,000** | **~9,600** | — |

| 方案 | 单次成本 (Quick+Deep) | 月均成本 (50 次/月) |
|------|---------------------|-------------------|
| DeepSeek v4 (flash+pro) | ¥0.05-0.10 | ¥2.5-5.0 |
| Qwen3 (flash+plus) | ¥0.03-0.08 | ¥1.5-4.0 |

| 对比: Claude (sonnet+opus) | $0.15-0.40 | ¥550-1,440 |

> 国产模型成本约为海外模型的 **1/100 ~ 1/300**。

## 8. 总结

### 核心结论

1. **DeepSeek v4 + Qwen3.7 是当前最适配本项目 A 股分析场景的国产模型组合**
   - DeepSeek v4：已验证适配、思维链可解释、极致性价比
   - Qwen3.7：中文金融理解最强、结构化输出最稳定、阿里云 SLA 保障

2. **GLM-5 作为第二梯队推荐**，推理能力顶级但平台生态略逊

3. **百度文心/讯飞星火/百川暂不推荐**，投入产出比不佳

4. **接入成本极低**：Qwen/GLM/MiniMax 均兼容 OpenAI API 格式，仅需扩展枚举 + 配置默认 Base URL

5. **国产模型成本优势巨大**：单次分析成本仅为海外模型的 1/100 ~ 1/300，是项目大规模使用 AI 分析的前提
